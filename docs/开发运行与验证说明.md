# 开发运行与验证说明

更新时间：2026-06-18

## 1. 当前实现范围

当前工程已经搭建为 TypeScript workspace：

```text
apps/mobile    Android Expo 应用骨架
apps/server    本地 Fastify + SQLite 服务端
apps/web       电脑 Web 管理与学习界面
packages/shared 共享类型、CSV 工具、SRS 复习算法
```

已实现的基本可用能力：

- Web 端连接本地服务并显示复习台、词库、CSV 导入导出、资源包和同步诊断。
- 服务端使用 SQLite 文件库存储单词、词义、音频元数据、卡片、复习记录、同步设备和资源包。
- 支持新增、删除、搜索单词。
- 支持 CSV 导入导出。
- 支持到期复习队列和 `again / hard / good / easy` 复习结果。
- 支持统计：总词数、待复习、今日复习、保持率、缺失音频。
- 支持局域网协议入口：`/health`、`/.well-known/elsync`、`/pair`、`/sync/changes`、`/sync/apply`。
- Android Expo 应用具备离线 SQLite 建表、新增单词、复习、连接电脑端健康检查的第一版骨架。

## 3. 常用命令

```bash
npm run typecheck
npm test
npm run build
```

导入基础词汇与本地发音资源包：

```bash
npm run import:basic-vocabulary -- --limit 300
```

从外部词表生成词书资源包。下面示例会读取 CEFR-J Vocabulary Profile 1.5 的 A1 词条，保存 200 个带本地发音的词到 `assets/resource-packs/cefr-j-a1-starter-2026-06-18/`，并登记到本地 SQLite 的 `resource_packs`：

```bash
npm run import:vocabulary -- \
  --limit 200 \
  --pack-slug cefr-j-a1-starter-2026-06-18 \
  --pack-name "CEFR-J A1 Starter Vocabulary" \
  --source-id cefr-j-a1-dictionaryapi \
  --word-list-url https://raw.githubusercontent.com/openlanguageprofiles/olp-en-cefrj/master/cefrj-vocabulary-profile-1.5.csv \
  --word-column headword \
  --filter-column CEFR \
  --filter-values A1 \
  --source-name "CEFR-J Vocabulary Profile 1.5" \
  --source-url https://github.com/openlanguageprofiles/olp-en-cefrj \
  --source-license "Free use for research and commercial purposes with proper citation; copyright Tono Laboratory at TUFS" \
  --tag cefr-j,a1,starter
```

如果第三方词典或发音服务限流，可以先用词表-only 模式下载常用词书。该模式不抓在线发音，但会尽量保留源词表中的中文释义、音标、英文释义和例句：

```bash
npm run import:vocabulary -- \
  --word-list-only \
  --limit 1000 \
  --pack-slug common-cet4-core-2026-06-18 \
  --pack-name "CET4 Core Vocabulary" \
  --source-id sholvoir-cet4-wordlist \
  --word-list-url https://raw.githubusercontent.com/sholvoir/vocabulary/main/origin/CET4-1.txt \
  --source-name "sholvoir/vocabulary CET4-1" \
  --source-url https://github.com/sholvoir/vocabulary \
  --source-license "MIT; original data sources documented in sholvoir/vocabulary README" \
  --tag sholvoir,cet4,exam,core
```

已用同一模式生成：

- `common-cet4-core-2026-06-18`：CET4 Core Vocabulary，1000 词，0 个音频。
- `common-cet6-core-2026-06-18`：CET6 Core Vocabulary，1000 词，0 个音频。
- `common-ielts-core-2026-06-18`：IELTS Core Vocabulary，1000 词，0 个音频。
- `common-toefl-core-2026-06-18`：TOEFL Core Vocabulary，1000 词，0 个音频。
- `common-gre-core-2026-06-18`：GRE Core Vocabulary，1000 词，0 个音频。

补指定词的发音资料：

```bash
npm run import:basic-vocabulary -- --words persist,verify
```

启动构建后的本地服务：

```bash
npm --workspace @els/server start
```

如果需要让服务在 Codex 临时命令结束后仍然保持可访问，使用 macOS 用户级服务托管：

```bash
launchctl remove com.yanghuanyu.english-learning-system.server >/dev/null 2>&1 || true
launchctl submit -l com.yanghuanyu.english-learning-system.server \
  -o /tmp/英语学习系统-server.log \
  -e /tmp/英语学习系统-server.err.log \
  -- /bin/zsh -lc 'cd "/Users/yanghuanyu/codex/English Learning System" && exec node --no-warnings apps/server/dist/index.js'
```

停止托管服务：

```bash
launchctl remove com.yanghuanyu.english-learning-system.server
```

检查服务状态：

```bash
launchctl list | grep com.yanghuanyu.english-learning-system.server
curl http://127.0.0.1:5174/health
```

服务端默认监听：

```text
http://127.0.0.1:5174
http://本机局域网 IP:5174
```

启动 Web 开发服务：

```bash
npm --workspace @els/web run dev
```

启动 Android Expo 开发服务：

```bash
npm --workspace @els/mobile run dev
```

连接 Android 真机或模拟器后运行：

```bash
npm --workspace @els/mobile run android
```
